## 穿透速度慢，为什么锅不在 frp？

先讲清楚一个基本概念，避免你和我当初一样方向搞错了。

**内网穿透（frp/NAT穿透）的工作原理**：你在家里的设备（比如群晖 NAS、树莓派）没有公网 IP，所以从外网无法直接访问它。frp 的做法是：在一台有公网 IP 的 VPS 上跑服务端（frps），家里设备跑客户端（frpc），外部访问请求先到达 VPS，再由 VPS 转发到你家里的设备。

问题来了。数据走的路是这样的：

1. 外网用户请求 → VPS（搬瓦工服务器，在美国洛杉矶）
2. VPS → 通过国际线路 → 你家路由器
3. 数据 → 家里 NAS → 原路返回

这意味着什么？哪怕你家的宽带是 500M，哪怕搬瓦工服务器本身带宽够宽，决定这整条链路速度的，是**国际线路质量**。用低端线路的机房做穿透，晚高峰国际出口拥堵，穿透速度就跟着一起塌。

这个道理搞清楚之后，接下来所有的选择就都有据可依了。

## 搬瓦工线路分级：穿透场景下哪条线最值钱

搬瓦工目前有几条主要线路，从穿透角度排一下优先级：

**第一档：香港 CN2 GIA**
物理距离最近，延迟可以低到 40-50ms 左右。流量走全程 CN2 GIA 优化线路，几乎没有绕路。做穿透的话，访问家里的设备就像访问一个国内服务器，体感差距很小。缺点很明显：价格贵，$89.99/月起步，不是人人能接受的。

**第二档：CN2 GIA-E（洛杉矶 DC6/DC9）**
美国西海岸到国内，电信线路平均延迟在 160-200ms 区间，三网回程都走 CN2 GIA，晚高峰丢包率接近 0。这是搬瓦工口碑最稳定的中端档位，做穿透跑日常业务（SSH 远程、轻量级文件访问、NAS 管理）完全够用，视频流转会因为延迟有点卡，但本地上传速度不差的话，200ms 延迟内的穿透体验已经算优秀。

**第三档：CN2 GT（DC3/DC8）**
延迟比 GIA 高一些，白天稳定，晚高峰国际出口可能出现拥堵，丢包率会上升。如果只是偶尔远程登录 SSH 查个东西，够用。如果想长期稳定跑穿透服务、看家里 NAS 的视频，高峰期体验会打折扣。

**第四档：普通 KVM（弗里蒙特、新泽西等）**
普通线路，延迟高，不走 CN2 优化。价格便宜，但穿透速度比较玄学——好的时候还行，差的时候基本没法用。

结论直接说：**做穿透，预算有限选 CN2 GIA-E；对速度有要求、家里主要是电信宽带的，直接上香港机房**。

## 速度慢的排查步骤（按照顺序来）

这部分把我自己踩过的坑和最后找到的解法整理成步骤，比网上那些"换机房"的万金油建议细一些。

**第一步：确认是线路问题还是本地网络问题**

在家里电脑上 ping 一下搬瓦工不同机房的测试 IP，看延迟和丢包。如果 ping 值跟平时相比差距不大，问题在其他地方；如果 ping 值高到几百毫秒或者丢包严重，就是线路问题。

顺便检查一下本地宽带是否正常。长城宽带、鹏博士这类小运营商，到国外的线路天然差，换成电信/联通/移动主流宽带才有后续优化的意义。

**第二步：开 BBR**

这步很多人忽略。搬瓦工 KVM 套餐默认不一定带 BBR，重装系统时选带 BBR 字样的镜像（比如 "CentOS BBR"），或者手动安装 BBR 脚本。BBR 是 Google 开发的 TCP 拥塞控制算法，对跨国链路的速度提升非常明显——尤其是在有一定丢包率的线路上，BBR 比默认算法表现好得多。

开了 BBR 之后再测，frp 穿透的下载速度通常会有明显提升。

**第三步：切换机房**

KVM 套餐默认可以在后台 KiwiVM 面板里切换机房，不需要重新购买。如果当前机房速度差，按这个顺序试：DC8 CN2 → DC3 CN2 → DC4 → DC9（需要先升到 CN2 GIA 套餐）。不同地区、不同运营商适合的机房不一样，多试几个总能找到速度最好的。

**第四步：frp 配置优化**

frp 本身也有几个影响速度的参数：

1. `use_compression = true`：开压缩，对文本类数据传输帮助大，但视频/图片这类已压缩的格式帮助有限，反而增加 CPU 消耗
2. `use_encryption = true`：开加密会消耗性能，如果对安全要求不是极高，可以暂时关掉测速对比
3 `pool_count`：连接池数量，适当调大（比如改成 10）可以减少新连接建立的延迟

测试速度的时候建议先把 compression 和 encryption 都关掉，单独测裸速，再按需开启。

**第五步：考虑升级套餐**

说实话，如果做穿透，普通 KVM 套餐的上限就在那里，改 frp 配置也突破不了线路的瓶颈。

预算有限的话，CN2 GIA-E 的限量版套餐是性价比最高的选择，价格有时候能到 $49.99/年，但要守库存。常规版 CN2 GIA-E 是 $169.99/年，折合每个月约 $14，穿透速度稳定，晚高峰不会崩。

**场景二：外网访问家里 NAS 看视频**

延迟影响不大，主要看带宽和线路稳定性。CN2 GIA-E 在晚高峰丢包接近 0 的情况下，流量跑到家里宽带上传速率上限是没问题的。普通 KVM 高峰期丢包会导致视频卡顿，体验差很多。

**场景三：内网穿透跑低延迟应用（远程桌面、游戏联机）**

这个对延迟要求最苛刻。洛杉矶 CN2 GIA 的 200ms 延迟，用来跑远程桌面实际操作感受是"有点卡但能接受"，打反应要求高的游戏就难了。想要低延迟，只能选香港机房（延迟 40-50ms），或者放弃搬瓦工做穿透，考虑其他国内转发方案。

稳定性优先，带宽其次。CN2 GIA-E 的 0 丢包表现在这个场景里很有价值。搬瓦工做了十几年，这类用途口碑一直不错。30 天内不满意可以全额退款，风险可控。

## 常见问题

纯粹从穿透速度来说，国内 VPS（阿里云/腾讯云轻量）做穿透延迟更低，因为数据不用绕国际线路。但国内 VPS 按流量计费，穿透流量大了费用蹭蹭涨；搬瓦工是固定月流量，适合流量需求稳定的场景。另外，有些特殊用途只能走境外 VPS，那就没得选了。

**Q：限量版套餐和常规套餐穿透效果一样吗？**

**Q：搬瓦工支持 frp 吗？有没有端口限制？**
